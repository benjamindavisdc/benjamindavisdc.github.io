Serverless Datafeed Pipeline

Stay up to date on the latest info with Python and DynamoDB.

X-ray
          
import requests
import base64
import boto3
            
def lambda_handler(event, context):
    # Scrape the webpage
    url = "https://api.github.com/repos/*"
    response = requests.get(url)
                
    if response.status_code == 200:
        page_text = response.json()["content"]
        decoded_bytes = base64.b64decode(page_text)
        scraped_text = decoded_bytes.decode(errors='replace')
            
        print("Scraped webpage text:")
        print(scraped_text)
                    
        # Put the scraped text into DynamoDB
        dynamodb = boto3.resource('dynamodb', region_name='us-east-1')
        table = dynamodb.Table('x-ray_table')
            
        response = table.put_item(
            Item={
                'file_path': url,
                'file_content': scraped_text
            }
        )
            
        return {
            'statusCode': 200,
            'body': 'Item added successfully to DynamoDB table.'
        }
    else:
        return {
            'statusCode': response.status_code,
            'body': 'Failed to scrape webpage.'
        }

'X-ray App'

Dynamic Content

Using Python and DynamoDB, we can deliver dynamic content to our front end, keeping us up to date on all infrastructure changes.

The X-ray app is an exciting project I developed to showcase my DevOps skills in infrastructure deployment. As a seasoned professional in this field, I realized that traditional portfolio websites were limited in demonstrating the backend aspects of my work. So, I set out to create a solution that allows visitors to delve into my infrastructure code and gain a deeper understanding of my expertise.

My primary motivation behind building the X-ray app was to address the need for a platform that highlights the most recent versions of my infrastructure files. With this app, I wanted to ensure that potential employers or clients can explore my code and stay updated on the latest developments in my projects. By dynamically pulling the code directly from my repositories, I eliminate the hassle of manually updating snippets and ensure that viewers always have access to the most up-to-date information.

DevOps

Another critical aspect I wanted to address with the X-ray app was the ability to iterate rapidly on new changes to my infrastructure. I found it cumbersome and time-consuming to manually update code snippets on my portfolio website whenever I made updates to my projects. To overcome this hurdle, the X-ray app leverages automation to retrieve the latest code from my repositories. This way, I can focus on refining and improving my infrastructure without worrying about keeping the website in sync.

Embracing Agility By building the X-ray app, I've created a platform that aligns with my development philosophy of embracing agility. It allows me to showcase my backend scripts, like Dockerfiles and Jenkinsfiles, in an organized manner, enabling visitors to navigate and understand my deployment processes effortlessly. The app provides an interactive and user-friendly experience, giving viewers a glimpse into the inner workings of my projects.

In summary, the X-ray app is my solution for showcasing my DevOps skills in infrastructure deployment. It ensures that viewers have access to the most recent versions of my infrastructure files, while also enabling me to iterate rapidly on new changes. By offering this transparency into my work, I aim to provide potential employers and clients with a comprehensive understanding of my expertise and commitment to delivering robust and scalable infrastructure solutions.
