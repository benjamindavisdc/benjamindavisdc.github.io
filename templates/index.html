1. Automated Cloud Infrastructure Blueprint

Infrastructure as Code

The first step I went through when creating my site is making a plan for the Backend infrastructure.

I used Terraform, AWS CodeCommit, AWS CodePipeline, and YAML scripting to create an infrastructure deployment pipeline.

This blueprint contains my main.tf file, sg.tf, and lambda.tf files as well as SSH keys and other supporting files.

The code is stored in a code repository on AWS and is configured to deploy automatically when updated. This folder is how I control my compute, database, security, and web app infrastrucutre.

2. CI/CD Container Deployment Pipeline

After deploying my server, it needs to be configured to host containers.

I built bootstrap scripts, Dockerfiles, and Jenkinsfiles to automate important tasks:

> Install Docker,
> Install Jenkins,
> Install Apache,
> Build a Docker Network,
> Build a Jenkins Image and Container,
> Build an Apache Image and Container,
> Intialize Jenkins,
> Initialize Apache,
> Configure the Jenkins Pipeline job from a Jenkinsfile,
> Deliver repo files to Apache web server directory.

3. X-Ray App - A Full Stack Python REST API

Dynamic Content

How do you show off your backend infrastructure over the internet?
With a unique full-stack app.

The final step is the most important! I built this site, along with a Python app to help me display all the backend code you will see here.

The Xray-App is a two part process that scrapes repositories, stores the data in DynamoDB, then displays the data here as HTML using Python Flask.

I realized that traditional portfolio websites were limited in demonstrating the backend aspects of my work. So, I set out to create a solution that allows visitors to delve into my infrastructure code and gain a deeper understanding of my expertise.

Services

In this project, I used Docker to create separate containers inside my server to handle both the Pipeline, and Web Hosting in a single VM.

Terraform allowed me to keep track of all the resources I've built for this project, and ensures that I have everything I need.

I used Jenkins to make sure my web server always stays up to date with the latest changes.

I manage my Apache server through the bash scripts in part 2.

EC2, ALB, ASG, Firewalls, Policies.
Working in AWS makes it easier for various services to communicate and create pipelines.

I stored my backend code in DynamoDB and brought it to the front end for easy visualization.

Lambda is an easy way to monitor code changes and update my code stored in the database.

My script specifies which files I want, and how they should be stored.

Flask turned my entire website into an app, integrating dynamic content into the HTML.
